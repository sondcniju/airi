+# Architecture: Gateway Security Hardening
+
+This document outlines the security audit results and the subsequent hardening of the AIRI WebSocket gateway (port `6121`).
+
+## 1. Problem Statement
+The AIRI Gateway is the central orchestration hub for all modules (Consciousness, Hearing, Speech, etc.) and external adapters (Discord Bot, VS Code).
+A security audit identified that:
+- **Default Binding**: The gateway bound to `0.0.0.0` (all interfaces), potentially exposing the WebSocket port to local networks or the internet in DMZ/port-forwarding scenarios.
+- **Lax Authentication**: Peers could connect without a token if no `authToken` was configured on the server, which was the default state.
+- **Incomplete Gating**: Even with a token, some event types were not strictly gated behind the `authenticated` state in the WebSocket handler.
+
+## 2. Implemented Solutions
+
+### 2.1. Local-Only Binding
+The gateway now defaults to binding strictly to `127.0.0.1` (localhost). This ensures that by default, only processes running on the same machine can reach the gateway.
+
+### 2.2. Mandatory `authToken` Handshake
+Authentication is no longer optional. Every peer must perform the `module:authenticate` handshake before announcing itself or interacting with other modules.
+- **Server Enforcement**: The server rejects any event (except heartbeats and authentication) if the peer is not in an `authenticated` state.
+- **Persistence**: The desktop app (`stage-tamagotchi`) generates a cryptographically secure `uuid`-based **App Key** on first run and persists it in `config.json`.
+
+### 2.3. Client-Side Handshake
+The `@proj-airi/server-sdk` was updated to automatically perform the handshake if a `token` is provided in the client constructor. All official integrations (Discord, VS Code) have been updated to support this parameter.
+
+## 3. Configuration & UI
+
+### Desktop App (`stage-tamagotchi`)
+The connection settings are now exposed in the UI under **Settings > Connection**:
+- **Hostname**: Defaults to `127.0.0.1`.
+- **App Key**: The unique key generated for this installation.
+- **Storage**: Persisted in the user data directory under `server-channel/config.json`.
+
+### Reconfiguring External Modules
+External modules must be updated to use the generated **App Key**:
+
+#### Discord Bot
+Set the `AIRI_TOKEN` environment variable to the App Key value.
+
+#### VS Code Extension
+Update the `AIRI: App Key` (`airi-vscode.authToken`) setting in VS Code with the App Key from the Desktop app.
+
+## 4. Handshake Protocol Details
+
+The handshake consists of a `module:authenticate` event sent immediately after connection:
+
+```json
+{
+  "type": "module:authenticate",
+  "data": {
+    "token": "YOUR_APP_KEY"
+  }
+}
+```
+
+The server responds with:
+- `registry:modules:authenticated`: Success.
+- `registry:modules:error` (invalid token): Failure; the connection will be gated and subsequent messages ignored.
+
+---
+**NOTICE**: This change is **breaking** for any third-party modules that do not implement the `module:authenticate` handshake. Please update your implementation to use the latest `@proj-airi/server-sdk`.

+# Gateway & Security Guide
+
+The AIRI Gateway is the "brain" that connects all parts of the application. To ensure your data remains private and secure, specifically when running in complex network environments, we have implemented several security hardening measures.
+
+## What's Changed?
+
+Recent updates have introduced two major security requirements:
+1.  **Local-Only Access**: By default, the gateway now only listens on `127.0.0.1` (your own computer). External devices cannot connect to it unless explicitly configured.
+2.  **Mandatory App Key**: Every connection to the gateway now requires a unique **App Key**. This prevents unauthorized apps from eavesdropping on your conversations or controlling your character.
+
+## Finding your App Key
+
+You can find your unique App Key and Hostname in the Desktop application:
+
+1.  Open the **Desktop App**.
+2.  Go to **Settings** (Gear icon).
+3.  Select the **Connection** tab.
+4.  You will see your **Hostname** and **App Key** there.
+5.  Click the **Copy** button next to the App Key to copy it to your clipboard.
+
+## Reconnecting Your Tools
+
+If your existing integrations (like the Discord Bot or VS Code) stopped working, you likely need to provide them with the new App Key.
+
+### VS Code Extension
+1.  Open VS Code.
+2.  Go to **Settings** (`Ctrl+,`).
+3.  Search for `AIRI: App Key`.
+4.  Paste your copied **App Key** into the field.
+5.  The extension will reconnect automatically.
+
+### Discord Bot
+Ensure your environment variables or configuration file includes the `AIRI_TOKEN` set to your **App Key**.
+
+### Other Adapters
+If you are using a custom adapter, ensure it sends the `module:authenticate` event immediately after connecting. If you are using the official `@proj-airi/server-sdk`, simply pass the `token` parameter to the `ServerClient` constructor.
+
+## Technical Storage
+
+The security settings are stored locally on your machine:
+- **Windows**: `%APPDATA%/airi/server-channel/config.json`
+- **Linux/macOS**: `~/.config/airi/server-channel/config.json` (typical)
+

+# Onboarding Overhaul: Easy vs. Advanced (The Sense Pivot)
+
+This plan outlines the restructuring of the initial AIRI setup experience to remove friction and highlight free-to-start services.
+
+## The Onboarding Flow
+
+### 1. Intro Screen
+The standard landing page with a warm welcome.
+
+### 2. The Choice (Fork Picker)
+A new screen presenting two distinct paths:
+*   **Easy Mode (Recommended)**: "I want to get started instantly with clear, easy-to-setup options."
+*   **Advanced Mode (Custom)**: "I want full control to configure my own providers and models."
+
+---
+
+## Path A: Easy Mode (The "Sense" Portal)
+
+This path replaces technical jargon with intuitive "Sense" terminology and focuses on providers with generous free tiers and no credit card requirements.
+
+### Simplified Setup Page
+*   **Consciousness** (formerly LLM)
+    *   **Provider**: **Qwen Portal**
+    *   **Interaction**: **[Authorize]** button (OAuth-style flow).
+    *   **Benefit**: Instant access with ~50 requests/day.
+*   **Speech** (formerly TTS)
+    *   **Provider**: **Deepgram Aura**
+    *   **Interaction**: **[API Key]** input box + signup link.
+    *   **Benefit**: Extremely low latency, high-quality voices.
+*   **Hearing** (formerly STT)
+    *   **Provider**: **Deepgram STT**
+    *   **Interaction**: **[API Key]** input box + signup link.
+    *   **Benefit**: $200 free credit (~50 hours of usage).
+
+> [!TIP]
+> **One Key, Two Senses**: Since Deepgram handles both Speech and Hearing, we will add verbiage explaining that the user can simply paste the same API key into both fields to finish the setup in one go.
+
+---
+
+## Path B: Advanced Mode (Legacy Flow)
+
+Maintains the granular control for power users who already have their preferred keys or local setups:
+1.  **Chat Picker**: Standard provider/API selection.
+2.  **Model Picker**: Detailed model settings.
+
+---
+
+## The Convergence (Final Steps)
+
+Both paths lead to:
+1.  **Character Picker**: Select the initial AIRI identity (e.g., Relu, Dr. Aria, or Lupin).
+2.  **Setup Complete**: Final success screen.
+

+- [ ] Update `i18n` strings to transition from LLM/TTS/STT to Consciousness/Speech/Hearing.
+- [ ] Update onboarding state machine logic to bridge the new paths.
